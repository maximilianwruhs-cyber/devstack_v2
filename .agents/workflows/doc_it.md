# Documentation Generator

1. Analyze the target file and understand the purpose, parameters, return types, and exceptions of all functions and classes.
2. Generate comprehensive and standardized documentation comments (e.g., JSDoc for TypeScript/JavaScript, Docstrings for Python) for all undocumented code blocks.
3. Update any outdated existing comments to reflect the current code structure.
4. Output only the updated code block with the comments injected. Do not provide conversational filler.
