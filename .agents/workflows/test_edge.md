# Edge Case Verifier

1. Review the existing logic for potential boundary issues, race conditions, type mishandling, and off-by-one errors.
2. Outline the expected failing scenarios or edge cases conceptually.
3. Automatically generate the unit tests to exercise these edge cases.
4. Execute the tests to verify the failing functionality, outputting the results and optionally a patch to resolve them.
