# Security Audit Workflow

1. Scan the specified file or project directory for common security vulnerabilities (e.g., hardcoded secrets, injection flaws, insecure dependencies).
2. Report any findings in a clear checklist format.
3. Automatically apply code changes to remediate the identified vulnerabilities without removing any existing functional logic.
4. If a vulnerability cannot be automatically fixed, explain why and provide specific steps the user must take to resolve it.
